# Contributing to UltimateTech Group Repositories

Thank you for your interest in contributing to UltimateTech Group. We maintain a private, security‑focused development environment. While this public repository serves as a corporate profile and documentation hub, contributions to our core software and infrastructure are managed through internal channels.

## Public Contributions (Documentation)

If you notice a typo in our public documentation, a broken link, or a formatting error, we welcome pull requests to correct these issues.

### How to Submit a Change

1. **Fork** the repository.
2. **Create a branch:** `git checkout -b fix/documentation-update`
3. **Commit your changes:** `git commit -m 'docs: correct contact link'`
4. **Push to the branch:** `git push origin fix/documentation-update`
5. Open a **Pull Request** against the `main` branch.

## Security & Code Contributions

Due to the proprietary and financial nature of our technology stack, we **do not** accept unsolicited code contributions (features, bug fixes, or security patches) via public GitHub pull requests.

If you are a security researcher and have identified a vulnerability, please refer to our [SECURITY.md](./SECURITY.md) policy for responsible disclosure.

## Contributor Code of Conduct

All interactions in this space are governed by our [Code of Conduct](./CODE_OF_CONDUCT.md). We expect professionalism and discretion.

## Internal Team Members

Internal developers should refer to the **UltimateTech Internal DevOps Handbook** (not hosted publicly) for access to private CI/CD pipelines and commit signing requirements.
